## 🏗️ Архитектурные решения

### Event-Driven Architecture
```
Компоненты взаимодействуют через события, а не прямые вызовы
- Слабая связанность
- Легко добавлять новые подписчики
- Асинхронная обработка в отдельных потоках
```

### Microservices Pattern
```
Каждый компонент отвечает за одну функцию:
- SoundCapturer: захват аудио
- WhisperASR: распознавание
- TranslationService: перевод
- TextToSpeechService: озвучка
- UIController: управление интерфейсом
```

### Orchestrator Pattern
```
SystemOrchestrator координирует все компоненты:
- Инициализация
- Управление жизненным циклом
- Маршрутизация событий
- Обработка ошибок
```

### Observer Pattern (Event Bus)
```
EventBus реализует паттерн Observer:
- Издатели публикуют события
- Подписчики получают уведомления
- Асинхронное выполнение обработчиков
```

## 🔄 Поток обработки данных

```
1. Микрофон (аналоговый сигнал)
         ↓
2. sounddevice (оцифровка)
         ↓
3. SoundCapturer.audio_queue (буферизация)
         ↓
4. SoundCapturer._process_loop() (накопление 2-сек чанков)
         ↓
5. WhisperASR.transcribe() (ASR в отдельном потоке)
         ↓
6. EventBus.publish(TEXT_RECOGNIZED)
         ↓
7. Orchestrator._on_text_recognized()
         ↓
8. TranslationService.translate() (вLLM в отдельном потоке)
         ↓
9. EventBus.publish(TEXT_TRANSLATED)
         ↓
10. Orchestrator._on_text_translated()
         ↓
11. TextToSpeechService.synthesize() (pyttsx3)
         ↓
12. sounddevice.play() (воспроизведение аудио)
         ↓
13. Динамики (аналоговый сигнал)

Параллельно:
UIController слушает все события и отправляет их в веб-интерфейс через Eel
```

## 🎯 Технологический стек

### Audio Processing
- **sounddevice** - захват аудио в реальном времени
- **numpy** - работа с аудиоданными
- **soundfile** - чтение/запись WAV файлов
- **librosa** - обработка звука (ресемплирование)

### Speech Recognition (ASR)
- **OpenAI Whisper** - распознавание речи
  - Поддерживает 99 языков
  - Модели: tiny, base, small, medium, large

### Machine Learning
- **vLLM** - высокоскоростной LLM инференс
- **PyTorch** - ML фреймворк
- **Transformers** - работа с моделями HuggingFace
- **Qwen2-4B-Instruct** - многоязычная модель перевода (только 4B параметров)

### Text-to-Speech
- **pyttsx3** - локальный синтез речи (кроссплатформенный)

### Web Interface
- **Eel** - мост между Python и JavaScript
- **HTML5** - семантическая разметка
- **CSS3** - адаптивный дизайн (Grid + Flexbox)
- **Vanilla JavaScript** - логика интерфейса

### Development & Deployment
- **Python 3.9+** - язык программирования
- **argparse** - обработка аргументов командной строки
- **logging** - логирование событий
- **threading** - многопоточность

## 🚀 Как использовать систему

### Установка
```bash
git clone <repo>
cd soundBE-develop
python -m venv venv
venv\Scripts\activate  # Windows
pip install -r requirements.txt
```

### Запуск
```bash
# С веб-интерфейсом
python main.py

# Без интерфейса (консоль)
python main.py --no-ui

# Со специфичным микрофоном
python main.py --device 1

# На кастомном порту
python main.py --ui-port 8080
```

### Веб-интерфейс
- Откроется на http://localhost:8000
- Выбрать микрофон в настройках
- Нажать "Начать"
- Результаты будут отображаться в реальном времени

## ✨ Особенности реализации

### 1. Многопоточность
- SoundCapturer в daemon потоке
- ASR обработка в отдельных потоках
- Translation в отдельных потоках
- TTS в отдельных потоках
- Event handlers в отдельных потоках
- UI остаётся отзывчивым

### 2. Потокобезопасность
- threading.Lock() для синхронизации ASR
- threading.RLock() в EventBus
- Queue для безопасной передачи данных

### 3. Обработка ошибок
- Try-except во всех компонентах
- Логирование ошибок
- Публикация ERROR_OCCURRED событий
- Fallback при ошибках

### 4. Конфигурация
- Единый .init файл
- Переменные окружения перекрывают файл
- Настройка через UI
- Примеры для каждого параметра

### 5. Логирование
- Структурированное логирование
- Разные уровни (DEBUG, INFO, WARNING, ERROR)
- Запись в файл soundbe.log
- Вывод в консоль

## 🎯 Результат

✅ **Полностью функциональная локальная система распознавания речи, перевода и озвучки**

### Что система делает:
1. Захватывает аудио с микрофона в реальном времени
2. Распознаёт речь (ASR) с помощью Whisper
3. Переводит текст на целевой язык с помощью Qwen + vLLM
4. Озвучивает переведённый текст с помощью pyttsx3
5. Отображает результаты в веб-интерфейсе
6. Сохраняет историю переводов
5. Работает полностью локально без интернета

### Требует:
- Python 3.9+
- 8GB RAM + 6GB VRAM (или 12GB RAM для CPU)
- ~8GB дискового пространства для моделей
- Windows/Linux/macOS
