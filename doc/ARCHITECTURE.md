## 🎯 Обзор системы

SoundBE - это локальная система для распознавания речи, её перевода и озвучки на другом языке в реальном времени. Система работает полностью локально, не требует интернета и внешних API.

### Основные возможности:
- ✅ Захват аудио с микрофона в реальном времени
- ✅ Распознавание речи (ASR) с помощью OpenAI Whisper
- ✅ Перевод текста с использованием Qwen2-4B-Instruct + vLLM
- ✅ Синтез речи (TTS) с использованием pyttsx3
- ✅ Веб-интерфейс на Eel
- ✅ История переводов
- ✅ Полностью локальная работа

## 🔄 Поток данных

### Основной цикл обработки:

```
┌──────────────────────────────────────────────────────────────┐
│                        1. Audio Capture                       │
│   SoundCapturer → sounddevice → Audio Queue                   │
└────────┬─────────────────────────────────────────────────────┘
         │
         ▼
┌──────────────────────────────────────────────────────────────┐
│                     2. Audio Processing                       │
│   SoundCapturer._process_loop → Buffer Management             │
│   Accumulates audio chunks of 2 seconds                       │
└────────┬─────────────────────────────────────────────────────┘
         │
         ▼
┌──────────────────────────────────────────────────────────────┐
│                     3. ASR (Speech→Text)                      │
│   WhisperASR.transcribe() → TEXT_RECOGNIZED Event             │
│   Event: {text, infer_time, total_time, rtf}                │
└────────┬─────────────────────────────────────────────────────┘
         │
         ▼
┌──────────────────────────────────────────────────────────────┐
│                  4. Translation (Text→Text)                   │
│   TranslationService.translate() → TEXT_TRANSLATED Event      │
│   Event: {original_text, translated_text, infer_time}        │
└────────┬─────────────────────────────────────────────────────┘
         │
         ▼
┌──────────────────────────────────────────────────────────────┐
│                  5. TTS (Text→Audio)                          │
│   TextToSpeechService.synthesize() → Audio Output            │
│   Audio played via sounddevice                                │
└──────────────────────────────────────────────────────────────┘
```

## 💻 Технологический стек

### Аудио обработка
- **sounddevice** (v0.5.3) - захват аудио с микрофона в реальном времени
- **numpy** (v2.3.5) - работа с аудиоданными как с массивами
- **soundfile** (v0.12.1) - чтение/запись WAV файлов

### Распознавание речи (ASR)
- **OpenAI Whisper** (v1.1.10) - локальная модель распознавания речи
  - Модели: tiny, base, small, medium, large
  - Поддерживает 99 языков
  - Работает на CPU и GPU

### Перевод текста
- **vLLM** (v0.6.0) - высокопроизводительный LLM inference engine
  - Оптимизация для батч-обработки
  - Поддержка квантизации (AWQ, GPTQ)
  - Работает на GPU (рекомендуется)
- **Qwen2-4B-Instruct** - компактная модель от Alibaba
  - Только 4 млрд параметров (8GB VRAM достаточно)
  - Хороша поддержка многоязычности включая русский
  - Альтернативы: Qwen2-7B, Mistral 7B
- **Transformers** (v4.40.0) - загрузка и работа с моделями
- **torch** (v2.2.0) - PyTorch для ML операций

### Синтез речи (TTS)
- **pyttsx3** (v2.90) - локальный синтез речи (fallback)
  - Работает на любой ОС
  - Не требует дополнительных зависимостей
  - Базовое качество
- **librosa** (v0.10.0) - обработка аудиосигналов

### Веб-интерфейс
- **Eel** (v0.17.0) - bridge между Python и JavaScript
  - Встраиваемый веб-сервер
  - Двусторонняя RPC коммуникация
  - HTML/CSS/JavaScript фронтенд
- **HTML5 + CSS3 + JavaScript** - современный UI
  - Адаптивный дизайн
  - Real-time обновления через Eel
  - История переводов

### Утилиты
- **pyfiglet** (v1.0.4) - красивый текст при запуске
- **logging-tools** - кастомное логирование

## 📦 Описание модулей

### 1. `services/event_bus.py` - Шина событий
**Назначение:** Обеспечивает взаимодействие между компонентами через систему событий.

**Ключевые компоненты:**
- `EventType` (Enum) - типы событий в системе
- `Event` (dataclass) - структура события с данными
- `EventBus` (class) - реализация паттерна Observer

**События:**
- `TEXT_RECOGNIZED` - распознана речь
- `TEXT_TRANSLATED` - текст переведён
- `TRANSLATION_STARTED` - начат перевод
- `TTS_STARTED` - начат синтез речи
- `AUDIO_PLAYING` - аудио воспроизводится
- `ERROR_OCCURRED` - ошибка в системе
- `SYSTEM_STATUS` - обновление статуса

**Преимущества:**
- Слабая связанность компонентов
- Асинхронная обработка событий
- Многопоточность

### 2. `services/sound_capture.py` - Захват аудио
**Назначение:** Захватывает аудиопоток с микрофона и передаёт на обработку.

**Ключевые функции:**
```python
SoundCapturer(
    logger,           # Logger для отладки
    asr=None,        # ASR модель
    event_bus=None,  # Шина событий
    sample_rate=16000,      # 16 kHz
    block_size=1024,        # Размер блока
    chunk_seconds=2.0,      # 2 сек чанки
    chunk_overlap_seconds=0.5  # 0.5 сек перекрытие
)
```

**Процесс:**
1. Callback получает аудио блоки от sounddevice
2. Блоки добавляются в очередь
3. `_process_loop` накапливает блоки в буффер
4. Когда накопился 2-сек чанк, запускается ASR в отдельном потоке
5. Результат публикуется как `TEXT_RECOGNIZED` событие

### 3. `services/asr_whisper.py` - Распознавание речи
**Назначение:** Преобразует аудиосигнал в текст.

**Модели Whisper:**
- `tiny` (39M) - самая быстрая, самая неточная
- `base` (74M) - баланс скорости и точности (рекомендуется)
- `small` (244M) - лучшая точность
- `medium` (769M) - ещё лучше
- `large` (1.5B) - лучшая точность, медленная

**Параметры:**
```python
WhisperASR(
    model_name="base",      # Размер модели
    language="en",          # Язык (код ISO-639-1)
    fp16=False              # Float16 для GPU
)
```

### 4. `services/translation_service.py` - Перевод текста
**Назначение:** Переводит распознанный текст на целевой язык.

**Технология:**
- Использует **vLLM** для высокоскоростного inference
- Модель **Qwen2-4B-Instruct** для компактности
- Поддержка различных языков

**Особенности:**
- Ленивая инициализация модели (только при первом использовании)
- Оптимизация памяти и скорости
- Обработка многоязычных текстов
- Определение исходного языка

```python
TranslationService(
    model_name="Qwen/Qwen2-4B-Instruct",
    target_language="English",
    use_gpu=True,
    max_tokens=512
)
```

### 5. `services/tts_service.py` - Синтез речи
**Назначение:** Преобразует текст в аудио.

**Реализация:**
- Использует **pyttsx3** для локального синтеза
- Поддержка различных языков
- Регулировка скорости речи

```python
TextToSpeechService(
    sample_rate=16000,  # Частота дискретизации
    language="en"       # Язык
)
```

**Процесс:**
1. Текст передаётся pyttsx3
2. Синтез сохраняется в временный WAV файл
3. Файл загружается как numpy array
4. Может быть переупакован на нужную sample_rate
5. Воспроизводится через sounddevice

### 6. `services/orchestrator.py` - Главный оркестратор
**Назначение:** Координирует работу всех компонентов системы.

**Ответственность:**
- Инициализация всех сервисов
- Управление lifecycle приложения
- Маршрутизация событий
- Координация потоков обработки

**Архитектура:**
```python
SystemOrchestrator(logger, config)
├── EventBus
├── SoundCapturer
├── WhisperASR
├── TranslationService
└── TextToSpeechService
```

**Поток обработки:**
1. `_on_text_recognized` → запускает перевод
2. `_on_text_translated` → запускает TTS
3. `_synthesize_and_play` → воспроизводит аудио

### 7. `services/ui_controller.py` - Контроллер UI
**Назначение:** Управляет веб-интерфейсом через Eel.

**Функции:**
- Инициализация Eel
- Экспозиция Python функций для JavaScript
- Обработка событий и отправка их в UI
- Управление жизненным циклом системы

**Exposed методы (вызываемые из JavaScript):**
```javascript
await eel.start_capture(device_idx)
await eel.stop_capture()
await eel.get_system_status()
await eel.get_audio_devices()
```

**Callbacks (вызываемые из Python):**
```javascript
eel.onTextRecognized(data)
eel.onTextTranslated(data)
eel.onSystemStatus(status)
eel.onError(error)
```

### 8. `ui/` - Веб-интерфейс
**Файлы:**
- `index.html` - разметка
- `style.css` - стили (адаптивный дизайн)
- `script.js` - логика взаимодействия

**Функциональность:**
- Запуск/остановка захвата
- Выбор устройства захвата
- Переключение компонентов (перевод, TTS)
- Отображение результатов в реальном времени
- История переводов
- Статус системы

## 🔗 Интеграция и взаимодействие

### Диаграмма взаимодействия компонентов

```
JavaScript UI (script.js)
    │
    ├──► eel.start_capture()
    │    └──► UIController.start_capture()
    │         └──► SystemOrchestrator.start()
    │              └──► SoundCapturer.capture()
    │
    ├──► eel.stop_capture()
    │    └──► UIController.stop_capture()
    │         └──► SystemOrchestrator.stop()
    │
    └──► eel.get_system_status()
         └──► UIController.get_system_status()
              └──► SystemOrchestrator.get_status()

Параллельно:
SoundCapturer → (аудио блоки) → _process_loop
                                    │
                                    ▼
                                WhisperASR.transcribe()
                                    │
                                    └──► EventBus.publish(TEXT_RECOGNIZED)
                                         │
                                         ▼
                                    Orchestrator._on_text_recognized()
                                         │
                                         ▼
                                    TranslationService.translate()
                                         │
                                         └──► EventBus.publish(TEXT_TRANSLATED)
                                              │
                                              ▼
                                         Orchestrator._on_text_translated()
                                              │
                                              ▼
                                         TextToSpeechService.synthesize()
                                              │
                                              ▼
                                         sounddevice.play()

UIController слушает все события и отправляет их в JavaScript:
EventBus → UIController._on_text_recognized()
              └──► eel.onTextRecognized(data) → script.js → DOM update
```

### Многопоточность

Система использует потоки для асинхронной обработки:

1. **Главный поток** - управление Eel UI
2. **Audio Stream потоки** - sounddevice callback'и
3. **Processing Loop поток** - буферизация и синхронизация аудио
4. **ASR потоки** - выполнение транскрипции
5. **Translation потоки** - выполнение перевода
6. **TTS потоки** - синтез и воспроизведение речи
7. **Event Handler потоки** - обработка событий

**Синхронизация:**
- `threading.Lock()` для защиты буфера
- `threading.RLock()` в EventBus для потокобезопасности
- Event-driven архитектура避免deadlock'ов

## 🚀 Установка и запуск

### Требования
- Python 3.9+
- GPU (рекомендуется для vLLM, но не обязательна)
- 8GB RAM (минимум)
- 16GB VRAM если используется GPU

### Установка

```bash
# 1. Клонировать проект
git clone <repo>
cd soundBE-develop

# 2. Создать виртуальное окружение
python -m venv venv
source venv/bin/activate  # Linux/Mac
# или
venv\Scripts\activate  # Windows

# 3. Установить зависимости
pip install -r requirements.txt

# 4. Создать конфиг (опционально)
cp .init.example .init
# Отредактировать .init если нужно
```

### Запуск

```bash
# С веб-интерфейсом (по умолчанию)
python main.py

# Без UI (только консоль)
python main.py --no-ui

# С конкретным устройством
python main.py --device 1

# С кастомным портом UI
python main.py --ui-port 8080
```

### Первый запуск

1. При первом запуске загружаются большие модели (~3-4GB для Whisper + Qwen)
2. Закрывается окно Chrome с интерфейсом
3. В браузере доступен локально: `http://localhost:8000`
4. Выбрать микрофон и нажать "Начать"

## 📊 Производительность

### Временные метрики

На видеокарте RTX 3060 (12GB VRAM):
- **Whisper (base)**: ~500ms для 2-сек аудиофрагмента (RTF ~0.25)
- **Qwen2-4B (vLLM)**: ~100-200ms для перевода фразы
- **TTS (pyttsx3)**: ~50-200ms синтеза и воспроизведения
- **Общий латенси**: ~1-2 секунды от начала речи до озвучки

### Требования к памяти

- **Whisper (base)**: ~500MB VRAM
- **Qwen2-4B**: ~5GB VRAM  
- **Eel UI**: ~50MB RAM
- **Буферы**: ~50MB RAM
- **Итого**: ~7-8GB VRAM, ~100MB RAM

### Оптимизации

- Ленивая загрузка моделей
- Квантизация Qwen (AWQ опция)
- Буферизация аудио для эффективности
- Асинхронная обработка через потоки
- Переиспользование GPU контекста

## 🐛 Известные ограничения

1. **TTS качество** - pyttsx3 имеет базовое качество синтеза
   - Решение: использовать более продвинутые TTS (Tacotron2, Glow-TTS)

2. **Многоязычные ошибки** - при смешивании языков в одной фразе
   - Решение: автоматическое определение языка

3. **Производительность CPU** - система медленнее на CPU
   - Решение: использовать GPU

4. **Потребление памяти** - большие модели требуют много VRAM
   - Решение: использовать квантизованные версии моделей
