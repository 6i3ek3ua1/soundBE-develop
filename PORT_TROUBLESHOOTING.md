# Решение проблемы с занятыми портами в SoundBE

## Проблема
Ошибка: `[WinError 10048] Обычно разрешается только одно использование адреса сокета`

## Причины
1. Предыдущий экземпляр системы не был корректно остановлен
2. Другой процесс использует нужный порт
3. Система Windows не позволяет повторно использовать порт сразу после освобождения

## Решения

### 1. Ручное управление портами
```bash
# Проверить занятые порты
netstat -ano | findstr :8000

# Завершить процесс по PID
taskkill /PID <PID> /F

# Завершить все процессы python
taskkill /IM python.exe /F
```

### 2. Настройка порта через аргументы командной строки
```bash
# Запуск на конкретном порту
python main.py --ui-port 9000

# Автоматический поиск свободного порта
python main.py --auto-port
python main.py --ui-port 8000 --auto-port
```

### 3. Настройка через файл конфигурации (.init)
```
UI_PORT=8080
```

### 4. Переменные окружения
```bash
set UI_PORT=9000
python main.py
```

## Рекомендации
- Используйте `--auto-port` для автоматического разрешения конфликтов
- Избегайте системных портов (1-1023)
- Порты 8080, 8081 могут требовать прав администратора на Windows
- Используйте порты от 8000 и выше для локальной разработки
